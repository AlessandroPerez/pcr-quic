Expand description

AES-256-GCM encryption context with cached key schedule

This provides much faster encryption/decryption when using the same key for multiple packets, as the AES key schedule is computed only once.

Encrypt in-place with cached key schedule (fastest path)

Uses BoringSSL’s scatter-gather API for true in-place encryption. The tag is appended directly after the ciphertext. Buffer must have room for plaintext_len + 16 bytes (tag)

Returns the total output length (ciphertext + tag)
